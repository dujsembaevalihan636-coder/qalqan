# Qalqan (Қалқан — «щит»)

Веб-платформа, которая защищает молодёжь Казахстана от дропперских схем и
финансовых пирамид: проверяет подозрительные сообщения, показывает реальные
последствия и собирает коллективную карту активных мошеннических схем по стране.

Дизайн — тёмная система **Dala** (void-канвас + фиолетовый акцент, светофор
вердиктов), см. `DESIGN-3.md`.

## Быстрый старт

```bash
npm install
npm run dev
# → http://localhost:3000
```

Сайт **работает сразу, без ключей** — в демо-режиме: анализ идёт по эвристике, а
карта и народная база — на сид-данных из `lib/seed.ts`. Как только добавишь ключи,
включается настоящий Claude и общая база Supabase.

## Переменные окружения

Скопируй `.env.local.example` → `.env.local` и заполни:

| Переменная | Где живёт | Зачем |
|---|---|---|
| `ANTHROPIC_API_KEY` | только сервер | настоящий анализ через Claude |
| `ANTHROPIC_MODEL` | сервер | модель (по умолчанию `claude-sonnet-5`) |
| `NEXT_PUBLIC_SUPABASE_URL` | фронт+сервер | адрес проекта Supabase |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | фронт | публичный ключ (чтение) |
| `SUPABASE_SERVICE_ROLE_KEY` | **только сервер** | запись репортов в обход RLS |

> ⚠️ Ключ Anthropic и service-role Supabase используются **только** в
> `app/api/**/route.ts` и никогда не попадают в браузер.

## Supabase

1. Создай проект на supabase.com.
2. В SQL Editor выполни `supabase/schema.sql`, затем `supabase/seed.sql`.
3. Пропиши ключи в `.env.local` — приложение автоматически переключится с
   in-memory на Supabase.

## Структура

```
app/
  page.tsx              Главная: hero + Проверка (A) + Последствия (B) + альтернатива (F)
  map/page.tsx          Карта угроз (C) + народный реестр + репорт
  learn/page.tsx        Разбор (D): приёмы манипуляции + симулятор пирамиды
  parents/page.tsx      Родителям (E)
  api/
    analyze/            POST — анализ сообщения (Claude/эвристика) + запись репорта
    consequences/       POST — таймлайн последствий (ст. 232-1)
    check-account/      POST — проверка реквизита по народной базе
    report/             POST — ручной репорт схемы
    map/                GET  — агрегаты по регионам для карты
    flagged/            GET  — народный реестр (поиск)
components/             UI (Dala): Constellation, Checker, ThreatMap, PyramidSimulator…
lib/                    ai.ts (Claude+эвристика), store.ts (Supabase+fallback), seed, types, kz
supabase/               schema.sql + seed.sql
```

## Демо-флоу (для защиты)

1. Главная → вставить мошенническое сообщение → **красный** вердикт, флаги, чек-лист.
2. **«Показать последствия»** → таймлайн 5 000 ₸ → ст. 232-1 → до 7 лет.
3. **Карта угроз** → твоя проверка появилась точкой на карте Казахстана.
4. Финал: **«вместо развода»** → легальная подработка (модуль F).

## Честно

- Народная база и «проверка реквизитов» на демо работают по сид-данным, а не по
  гос-API (его нет в открытом доступе).
- Qalqan — независимый инструмент, не официальный сервис АФМ.
- Продукт даёт ориентир и предупреждение, а не юридическую консультацию.
